Guide: How The Algorithms Work

Use this quick guide to understand the simulator workflow and the logic behind FIFO, LRU, and Optimal page replacement strategies.

How To Use The Simulator

  1. Open the Simulator page and set the number of frames.
  2. Enter the page reference sequence using spaces or commas.
  3. Choose one algorithm or select Compare All.
  4. Run the simulation to view faults, hits, and memory transitions.
  5. Analyze the bar chart and step tables to compare behavior.

FIFO

First-In-First-Out removes the oldest loaded page when memory is full. It is simple and fast but can produce more faults on certain patterns.

LRU

Least Recently Used replaces the page that has not been used for the longest time. It generally performs better than FIFO in practical workloads.

Optimal

Optimal replaces the page whose next use is farthest in the future. It provides the minimum possible faults for a known reference sequence.

Interpreting The Results

  • Page Fault: Requested page is not in memory and must be loaded.
  • Page Hit: Requested page is already present in memory.
  • Fault Ratio: Faults divided by total references.
  • Hit Ratio: Hits divided by total references.
  • Step Trace: Shows frames after every request and replacement events.
